Port Statistics Data Submission

This page explains what data is expected from ports and how the data can be submitted in two ways: with an Excel file or via the National Port API.

What Data Is Expected from Ports

The sections below summarize the minimum data expected from ports. Excel and API submissions have separate technical formats and validation rules.

1) Report Header Data (Required)

  • reporter: name of the reporting organization.
  • type: report type (for example, Port).
  • timeframeStart: start date and time of the reporting period.
  • timeframeEnd: end date and time of the reporting period.

2) Port Call Rows (At Least 1 Row)

  • Each port call row must include a vessel identifier.
  • A valid identifier can be one of the following: imoNumber, cfrNumber, eniNumber, mmsiNumber, callSign or otherId.
  • Additional details may include arrival and departure times, passenger data, and cargo data.

Option 1: Excel File (Non-Technical User)

If you are not using a technical integration, complete the Excel data according to the format documentation. This is the recommended option for non-technical users.

Excel Required Fields (from MLT2 format)

These fields must be available in the Excel content for successful processing:

  • Basic section: MLT-011 (must be "Port"), MLT-015 (version), MLT-010 (reporter), MLT-120 (reporting port/UN-LOCODE).
  • Port call section: MLT-100 (visit ID), MLT-130 (arrival date-time), MLT-131 (departure date-time), MLT-113 (ship name).
  • Vessel identification: at least one of MLT-110 (IMO), MLT-111 (MMSI), MLT-112 (call sign), MLT-114 (other identifier).
  • Traffic direction: MLT-191 (arrives in domestic/foreign), MLT-192 (departs in domestic/foreign).
  • Cargo rows: if inward/outward cargo rows are provided, all cargo fields in that row are required.

Excel File Rules and Validation Notes

  • File format must be .xlsx; maximum file size is 10 MB.
  • Main data must be in the first sheet.
  • In the human-fillable Excel template, support sheets for cargo groups and unit types are included as hidden sheets named Tavararyhmä and Kuljetusyksikkötyyppi.
  • For automatically generated Excel files, the support sheets are not required by the parser. The first sheet must still contain the main port call and cargo data.
  • Date-time values should be valid Excel date/time values displayed as yyyy-mm-dd hh:mm, or ISO 8601 strings if generated as text.
  • Rows starting with Yhteensä in cargo tables are ignored by the parser.
  • If required fields are missing, the file is rejected with field-level validation errors.

Option 2: National Port API (Technical User)

If your port has integration capability, submit the data as a JSON message directly to the API endpoint.

The statistics submission endpoint is postStatistics (Ports)

  • HTTP method: POST
  • Content-Type: application/json
  • Authentication: Bearer JWT
  • Always submit reportHeaderData and at least one portCallInformation row.
  • Make sure each portCallInformation row has at least one valid identifier.

Minimum JSON Example

{
  "reportHeaderData": {
    "reporter": "Example Port Ltd",
    "type": "Port",
    "timeframeStart": "2026-05-01T00:00:00+03:00",
    "timeframeEnd": "2026-05-31T23:59:59+03:00"
  },
  "portCallInformation": [
    {
      "shipName": "EXAMPLE VESSEL",
      "mmsiNumber": "230123456"
    }
  ]
}

Excel to JSON Mapping (Most Important Fields)

  • MLT-010 (Reporter) -> reportHeaderData.reporter
  • MLT-011 (Report type "Port") -> reportHeaderData.type
  • MLT-130 / MLT-131 -> portCallInformation[].dateTimeOfArrival / dateTimeOfDeparture
  • MLT-113 (Ship name) -> portCallInformation[].shipName
  • MLT-110 / 111 / 112 / 114 -> portCallInformation[].imoNumber / mmsiNumber / callSign / otherId
  • MLT-410...450 and MLT-510...550 -> portCallInformation[].inwardCargo[] and outwardCargo[]

How to Choose the Submission Method

  • Choose Excel if data is filled in manually or no integration is available.
  • Choose API/JSON if data is produced automatically from port systems.
