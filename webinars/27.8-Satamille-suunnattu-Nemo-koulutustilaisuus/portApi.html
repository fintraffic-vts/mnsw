Introduction

Welcome to the documentation for the National API - Port API. This API is designed to streamline operations for Finnish ports, facilitating the efficient management of port calls, formalities, and environmental compliance.

The Port API offers a comprehensive suite of services for Finnish port companies, enabling them to post statistical data, access detailed information about port calls and associated formalities and can search and get ship information. This API is tailored to support the dynamic needs of port operations, promoting sustainability and efficiency.

Business logic

The business logic of the National Port API, as described in this document, revolves around providing a platform for port authorities and other maritime stakeholders. The core functionalities include:

  • Authentication and Authorization: Organisations must have a valid client ID and client secret to access the API. These credentials are obtained from the NEMO system and are essential for submitting and receiving data securely.

  • Data Submission and Retrieval: The API facilitates the submission of port statistic as JSON objects and the retrieval of port data and ship data responses from the NEMO system.

  • Compliance with EU Directive 2010/65/EU: The API supports formalities based on the directive, ensuring compliance with European Union regulations for maritime reporting along with national legislation and port authority regulations.

  • Localised Processes for Finland: While the API serves as a bridge to the European system, it also offers customization to meet the specific needs of Finnish maritime operations.

  • Real-Time Processing and Validation: Submitted data is validated by the NEMO system, with the API providing immediate feedback on errors or confirmation of successful submissions.

The National Port API’s business logic aims to streamline maritime reporting processes, enhance data security and compliance, and improve operational efficiency for maritime stakeholders in Finland and across Europe.

National Port API endpoints

You can find more detailed information about the Port API endpoints, as well as their requests and responses, in the technical documentation.

Components used in Port API

  • Port ERP: External ERP which makes a JSON call with valid JWT token

  • Ports: External REST API interface for port calls and MLT2 statistics provided by Port-api

  • Ships: External REST API interface for ships provided by Port-api

  • National Port API: National API route for port calls, statistics and ships

  • PermissionsApi: Internal interface for checking permissions provided by Nemo Ims

  • Nemo Ims: Nemo Identity management system

  • NationalAPIapi: Internal API provided by Nemo backend

  • Nemo Backend: Nemo core system

  • ShipAPIapi: Internal interface by Nemo ESD

  • Nemo ESD: Nemo ship database
